Guida alle classi di Bootstrap (livello base/intermedio)

Che cosa sono le classi più usate, come funzionano e come combinarle rapidamente per layout responsive e componenti standard.

Includere Bootstrap (rapido)

CDN (rapido, prototipo):

<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/css/bootstrap.min.css" rel="stylesheet" integrity="sha384-..." crossorigin="anonymous">
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.2/dist/js/bootstrap.bundle.min.js" integrity="sha384-..." crossorigin="anonymous"></script>

Per produzione preferisci installazione via NPM e compilazione Sass per personalizzare e ridurre il bundle.

Container, Grid e colonne

  • Container:
    • .container — larghezza fissa responsive
    • .container-fluid — piena larghezza
    • .container-{breakpoint} (es. .container-md) per container fluidi fino a un breakpoint
  • Grid — sistema a 12 colonne:
    • .row per riga
    • .col per colonne fluide
    • .col-6, .col-md-4 per larghezze esplicite e responsive

Esempio

<div class="container">
  <div class="row">
    <div class="col-12 col-md-6">Colonna A</div>
    <div class="col-12 col-md-6">Colonna B</div>
  </div>
</div>
Le classi responsive seguono lo schema .col-{breakpoint}-{n}, breakpoint: sm, md, lg, xl, xxl.

Utilities essenziali (rapido cheat sheet)

Spaziatura

m (margin), p (padding). Sintassi: m{t|b|s|e|x|y}-{0..5|auto}

.mt-3 .px-2 .mb-0 .mx-auto
Display & visibilità

.d-none .d-block .d-flex + responsive: .d-md-none .d-lg-flex

.d-none .d-md-block
Flex utilities

.d-flex .flex-row .flex-column .justify-content-between .align-items-center

.d-flex.justify-content-between.align-items-center
Tipografia

.text-center .text-md-start .text-muted .fw-bold

.text-primary .text-uppercase .lead
Colori & sfondi

.text-*, .bg-* (primary, success, danger, warning, info, light, dark)

.text-white .bg-primary
Immagini

.img-fluid (responsive), .rounded, .rounded-circle

<img src="..." class="img-fluid rounded" alt="...">
Ordinamento e allineamento

.order-1 .order-md-2 .align-self-start

Visually hidden

.visually-hidden per nascondere visivamente ma lasciare accessibile

Modal (struttura)

  </div>

Classi responsive (pattern)

Bootstrap usa breakpoints standard: sm (576px), md (768px), lg (992px), xl (1200px), xxl (1400px). Le utilities supportano questa sintassi:

.d-none .d-md-block        /* visibile da md in su */
.col-12 .col-md-6             /* colonna 100% mobile, 50% md+ */
.text-center .text-lg-start   /* testo centrato mobile, left su lg+ */
Quando usi utility responsive pensa mobile-first: definisci stili di base per mobile e applica modifiche con classi breakpoint.

JS integrato e Accessibilità

  • Bootstrap JS (bundle) include Popper per tooltip/popover: importa bootstrap.bundle.min.js.
  • I componenti usano data attributes per inizializzazione automatica (es. data-bs-toggle).
  • Accessibilità:
    • Navbar toggler: usa aria-controls, aria-expanded, aria-label.
    • Modal gestisce focus trap e aria-hidden automaticamente quando usi l'API/markup corretto.
    • Tooltip/popover: assicurati di fornire contenuti alternativi se necessari.

Personalizzazione & Ottimizzazione

1) Sass e variabili

Installa Bootstrap via NPM e usa Sass per sovrascrivere variabili prima di importare il core:

// _custom.scss
$primary: #0b71e5;
$body-font-family: 'Inter', system-ui, -apple-system, 'Segoe UI', Roboto, Arial;

@use "bootstrap" with ($primary: $primary, $body-font-family: $body-font-family);

2) Import selettivo

Importa solo i partial necessari per ridurre peso (grid, buttons, forms, utilities).

3) Tree‑shaking / PurgeCSS

Usa PurgeCSS per rimuovere classi non usate; safelist le classi generate dinamicamente (es. btn-${color}).

4) CDN con SRI

<link rel="stylesheet" href="https://cdn..." integrity="sha384-..." crossorigin="anonymous">

Pattern e Best Practice

  • Preferisci le utility per layout veloci e componenti Bootstrap per UI consistente.
  • Non sovrascrivere pesantemente il CSS base: usa variabili o crea classi derivanti (es. .btn--brand che estende .btn).
  • Separare styling custom in file separato importato DOPO bootstrap per garantire override semplici.
  • Evita di miscelare troppo le classi utility e le regole CSS personalizzate nello stesso markup: mantieni chiarezza semantica.
